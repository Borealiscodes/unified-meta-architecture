# **UMM Branches Reference**  
*(Public‑Facing • Flexible • Contextual • ND‑Accessible)*

Branches are the gentle extensions of a Tree.  
They offer variation without altering the trunk, and expression without changing the structure.

---

# 🌿 **1. Definition**

A Branch is an **optional, contextual extension** that grows outward from a Tree’s trunk.  
It provides an alternate articulation, tone, or pathway while preserving the Tree’s core logic.

Branches allow the same structure to be expressed in different ways depending on:

- audience  
- cognitive load  
- purpose  
- depth  
- pacing  
- context  

They are how a Tree adapts without changing.

---

# 🌳 **2. Role**

Branches serve as **parallel pathways** that help a Tree meet users where they are.

They support:

- simplified explanations  
- deeper technical dives  
- ND‑paced versions  
- visual or metaphor‑based versions  
- teaching‑oriented versions  
- tone‑matched versions  
- accessibility‑aligned versions  

Branches make the Tree more usable without altering its architecture.

---

# 🪵 **3. Constraints**

Branches are gentle and flexible, but they follow clear boundaries:

- A Branch **cannot modify the trunk**.  
- A Branch **cannot override Roots**.  
- A Branch **cannot merge with other Branches**.  
- A Branch **must remain reversible**.  
- A Branch **must remain optional**.  
- A Branch **must not introduce structural load**.  
- A Branch **must not create contradictions**.  

Branches express; they do not define.

---

# 🍃 **4. Types of Branches**

Branches come in several natural forms.  
Each serves a different purpose while remaining structurally light.

### **4.1 User‑Tailored Branch**  
Adjusts tone, pacing, or framing to match a specific user’s needs.

### **4.2 Beginner Branch**  
Simplifies concepts without losing accuracy.

### **4.3 High‑Altitude Meta Branch**  
Explains the system from a conceptual or architectural perspective.

### **4.4 ND‑Paced Branch**  
Uses gentle pacing, low ambiguity, and reduced cognitive load.

### **4.5 Visualization Branch**  
Expresses the Tree through diagrams, shapes, or spatial metaphors.

### **4.6 Teaching Branch**  
Structures the Tree for learning, instruction, or onboarding.

### **4.7 Narrative Branch**  
Uses story, analogy, or metaphor to convey the same structure.

### **4.8 Technical Branch**  
Provides deeper detail for advanced users without altering the trunk.

---

# 🌱 **5. Branch Behavior**

Branches behave like natural extensions:

- They grow outward, not downward.  
- They never carry the weight of the trunk.  
- They can be pruned, replaced, or regrown.  
- They can coexist without conflict.  
- They can be created as needed.  
- They can be retired when no longer useful.  

Branches are expressions, not commitments.

---

# 🌤 **6. When to Use a Branch**

A Branch is appropriate when:

- the user needs a different tone  
- the user needs a different depth  
- the user needs a different pacing  
- the user needs a different modality  
- the Tree is too dense for the moment  
- the context requires adaptation  
- the audience is varied  
- the purpose shifts (teaching, onboarding, public explanation)  

Branches are tools for accessibility, clarity, and kindness.

---

# 🍂 **7. When *Not* to Use a Branch**

A Branch should **not** be used when:

- the trunk itself needs revision  
- the user is asking for structural change  
- the adaptation would introduce contradiction  
- the adaptation would violate Roots  
- the adaptation would merge Trees  
- the adaptation would create load or confusion  

Branches are for expression, not architecture.

---

# 🌳 **8. Creating a New Branch**

A new Branch can be created when:

- a new audience emerges  
- a new modality is needed  
- a new teaching context appears  
- a new accessibility requirement arises  
- a new tone or depth is appropriate  

A Branch should be:

- light  
- reversible  
- optional  
- consistent with the trunk  
- aligned with Roots  
- clearly scoped  
- non‑structural  

Branches grow naturally when needed.

---

# 🍁 **9. Examples of Branches**

- A gentle ND‑paced explanation of membranes  
- A high‑altitude overview of the UMM Tree  
- A visual diagram of the ecological layers  
- A beginner‑friendly introduction to witness logic  
- A narrative metaphor explaining planes  
- A technical deep dive into membrane constraints  

Each expresses the same trunk in a different way.

---

# ⭐ **Summary**

Branches are the flexible, contextual extensions of a Tree.  
They adapt expression without altering structure, offering multiple pathways for understanding while preserving the integrity of the UMM ecosystem.
