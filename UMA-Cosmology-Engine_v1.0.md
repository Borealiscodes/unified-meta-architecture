# 1. Cosmology Engine Definition

The UMA Cosmology Engine is a structural module that models how conditions produce states, how states transition, and how those transitions converge toward clarity or fragmentation. It is not metaphysical; it is a mapping engine.

The engine provides:
- a topology (Forest Cosmology),
- a dynamics model (flows),
- a state model (conditions, states, forces),
- and a set of invariants ensuring non‑coercive, human‑rights‑aligned behavior.

# 6. Fork Patch (Center + Exit Invariants)

## Fork Definition
A **fork** is a structural divergence where an agent or topology attempts to override, monopolize, or falsify the center or exit‑conditions.

Homura’s Rebellion is the canonical fork case.

## Invariant 1 — Center Non‑Possession
The center cannot be owned, monopolized, or weaponized.  
Any attempt is classified as a distortion state.

## Invariant 2 — Exit‑Condition Integrity
Exit‑conditions cannot be removed from the topology.  
Any architecture that denies exits is coercive.

## Invariant 3 — Non‑Coercive Topology
No valid engine configuration may rely on coercion or captivity.

## Fork Handling Procedure
1. Classify fork type (center hijack, coercive topology).  
2. Witness Plane interprets as distortion.  
3. Boundary Plane identifies enabling conditions.  
4. Engine routes analysis back to non‑coercive configurations.
