# Module Index
   *The canonical index of all UMA modules, organized by plane and structural function.*
# 1. Purpose
The Module Index is the canonical index of all UMA modules.  
It lists modules by plane and structural function, without defining their internal logic.

This index is:
- non‑interpretive
- non‑narrative
- structurally descriptive only

It anchors where modules live inside UMA.

# 3. Cross‑Plane Module Relationships
This index does not define module internals.  
It records where modules belong and how they are grouped.

High‑level relationships:
- Boundary Layer modules constrain all other modules  
- Governance modules constrain capabilities of all modules  
- Structural modules define what modules may reference  
- Documentation modules record module definitions and changes  
- Developmental modules define when modules may activate  
- Dynamic modules define how modules behave over time  
- Integrative modules ensure module‑level coherence  

---

# 4. Operational Signature
The Module Index ensures:
- a single canonical list of modules  
- clear placement within planes  
- non‑overlapping structural roles  
- a stable reference for future module specifications  

It is the structural catalog of UMA’s internal components.
